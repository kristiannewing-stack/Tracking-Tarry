## Code of Conduct

This project is committed to providing a welcoming and inclusive environment for all contributors. We expect all participants to:

- Be respectful and considerate in communication
- Accept constructive criticism gracefully
- Focus on what's best for the community and project
- Show empathy towards other community members

## How Can I Contribute?

There are many ways to contribute to this project:

### 1. **Report Bugs**
Found a bug? Please open an issue with detailed information about the problem.

### 2. **Suggest Features**
Have an idea for a new monitored feature or improvement? We'd love to hear it!

### 3. **Improve Documentation**
Help us make the documentation clearer, fix typos, or add examples.

### 4. **Write Code**
Fix bugs, add new features, or improve existing functionality.

### 5. **Test on Different Systems**
Test the tool on different Windows versions and hardware configurations.

### 6. **Spread the Word**
Share the project with others who might benefit from it.

## Development Setup

To contribute to Tracking Tarry you will need:

-  Windows 11
-  Python (latest stable version recommended)
-  Git
-  A code editor such as VS Code
After cloning the repository, install any required dependencies and run the project locally to ensure everything is working before making changes.

## Contribution Guidelines 

When contributing code or documentation, please:
- Keep changes focused and relevant to the project
- Write clear and maintainable code 
- Update documentation if your changes affect usage 
- Test your changes before submitting them
- Ensure your code does not introduce unnecessary complexity

## Reporting Bugs

When reporting a bug, please include:
- A clear description of the issue 
- Steps to reproduce the problem 
- Expected behaviour
- Actual behaviour
- Your Windows version and system configuration if relevant
Providing detailed information helps us diagnose and fix issues faster.

## Suggesting Features

Feature suggestions are welcome. When suggesting a feature:
- Explain the feature clearly    
- Describe why it would be useful  
- Include any implementation ideas if possible
Please keep feature suggestions aligned with the project's goal of **transparency regarding Windows telemetry and background services**.

## Code Style

To keep the codebase consistent:
- Use clear and descriptive variable names
- Follow consistent indentation and formatting
- Write comments where necessary to explain complex logic
- Avoid unnecessary or overly complex code
Consistency helps keep the project maintainable.

## Commit Messages

Write clear and descriptive commit messages.

Examples:
Add monitoring for telemetry service  
Fix tray icon not updating correctly  
Update README with installation instructions

Good commit messages make the project history easier to understand.

## Pull Request Process

When submitting a pull request:

1. Ensure your code builds and runs correctly.
2. Provide a clear description of the changes. 
3. Reference any related issues if applicable.
4. Update documentation if necessary.
    

**All code must be reviewed by at least one other team member/main contributor before merging.**  
This ensures code quality, security, and consistency across the project.

## Project Philosophy

Tracking Tarry is designed to provide **transparency and information**, not to promote or discourage the use of Windows telemetry features.

Our goal is to:
- Inform users about background services and telemetry activity
- Present information in a clear and neutral way
- Promote awareness and understanding of system behaviour
Contributions should align with this philosophy.
